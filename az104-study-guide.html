Module 01

Identities & Access

Control who can log in to Azure and what they're allowed to do.

🔐 Azure Active Directory (Entra ID)

Think of it as the front door to Azure. Every user, app, or service needs an identity here to access anything in Azure.

Key objects

👤
Users
Individual accounts. Members (internal) or Guests (external, B2B).
👥
Groups
Assign permissions to many users at once. Security or M365 groups.
🤖
Service Principals
Identity for apps or automated scripts to authenticate.
🔑
Managed Identity
Auto-managed identity for Azure resources. No password to manage.

🛡 RBAC — Role Based Access Control

RBAC controls access to Azure resources. The formula: assign a role to a security principal at a scope.

Built-in roles you must know

RoleCan doCannot do
OwnerFull control + manage access—
ContributorCreate & manage all resourcesCannot assign roles
ReaderView everythingCannot change anything
User Access AdminManage access (roles)Cannot modify resources
✅ Real-world example
Your dev team has 10 people. Instead of setting permissions for each person: create a Group called Dev-Team, assign the Contributor role to that group at the Resource Group scope. Done — all 10 devs can manage resources there, and you only manage one assignment.
⚠️ Exam trap
RBAC controls access to Azure resources (portal, CLI, APIs). It does NOT control what happens inside the OS of a VM. For OS-level access, you use local accounts or Active Directory — totally separate.
Quick Check — Module 01
A developer needs to deploy resources in a resource group, but should NOT be able to grant permissions to others. Which role should you assign?
Module 02

Virtual Networking

Build isolated, secure networks for your Azure resources.

🌐 Virtual Network (VNet)

A VNet is your private network inside Azure. Resources inside the same VNet can talk to each other. By default, they cannot reach the internet or other VNets without extra configuration.

NSG — Network Security Group

An NSG is a firewall for your subnet or NIC. It has inbound and outbound rules. Each rule has a priority number — lower number = higher priority. First matching rule wins.

✅ Example NSG rules
Priority 100 — Allow — Inbound — Port 443
Priority 200 — Deny — Inbound — Port Any

Result: Only HTTPS traffic (port 443) gets in. Everything else is blocked by rule 200.

🔗 Connectivity services

ServiceWhat it doesKey detail
VNet PeeringConnect two VNets directlyNot transitive by default
VPN GatewayEncrypted tunnel over internet to on-premUp to 10 Gbps, slower setup
ExpressRoutePrivate dedicated link to AzureNot over internet, higher SLA
Azure Load BalancerDistribute traffic across VMsLayer 4 (TCP/UDP), public or internal
Application GatewayHTTP load balancer with routingLayer 7 — can route by URL path
Azure DNSHost DNS zones in AzurePrivate zones for internal resolution
⚠️ Peering trap
VNet peering is not transitive. If A peers with B, and B peers with C — A cannot talk to C automatically. You must explicitly peer A↔C as well. This comes up a lot in exam scenarios.
Quick Check — Module 02
You have two NSG rules: Priority 100 — Deny port 22, and Priority 200 — Allow port 22. What happens when someone tries to SSH (port 22)?
Module 03

Virtual Machines

Compute in the cloud — deploy, size, protect, and back up VMs.

🖥 VM basics

An Azure VM is a computer running in Microsoft's data center. You choose the OS (Windows / Linux), size (CPU + RAM), and disk type. You pay per minute while it's running.

VM size families

FamilyBest forExample
B-seriesLow cost, bursty workloadsDev/test, small apps
D-seriesGeneral purpose (balanced CPU/RAM)Web servers, databases
E-seriesMemory optimizedIn-memory databases, SAP
F-seriesCompute optimizedBatch processing, gaming
N-seriesGPU workloadsMachine learning, rendering

🏗 Availability options

If a VM crashes or the hardware fails, you want your app to stay up. Azure offers three levels of protection:

OptionProtects againstSLA
Availability SetHardware failures within one data center (fault/update domains)99.95%
Availability ZonesFull data center failure (different buildings)99.99%
VM Scale SetsTraffic spikes — auto-scales VM count up and down99.95%

💽 VM disks

🟢
OS Disk
System drive (C: or /). Persistent. Required.
🔵
Data Disk
Extra storage you attach. Persistent. Optional.
🔴
Temp Disk
Fast local storage. LOST when VM is deallocated!
⚠️ Temp disk trap
Temp disk data is wiped when you stop/deallocate a VM (not just restart). Never store anything important there. It's only for swap files or temporary processing.

Backup

Use Azure Backup with a Recovery Services Vault to back up VMs. You define a backup policy (frequency + retention). You can restore a full VM or individual files/folders.

Quick Check — Module 03
Your web app is hosted on Azure VMs. You need the app to survive a complete data center failure. What should you use?
Module 04

Storage

Scalable, durable cloud storage for any type of data.

💾 Storage services

ServiceUse it forReal example
Blob StorageUnstructured files (images, videos, logs, backups)Profile pictures, VM backups
Azure FilesFile shares (SMB/NFS) — mount as a network driveShared drive for multiple VMs
Table StorageNoSQL key-value data, simple and cheapSession data, device telemetry
Queue StorageMessage queuing between servicesDecouple app components

🌡 Blob access tiers

Choose a tier based on how often you access the data. Lower access frequency = lower storage cost but higher retrieval cost.

💡 Lifecycle management
You can create a lifecycle policy to automatically move blobs between tiers. Example: move to Cool after 30 days of no access, move to Archive after 90 days. This saves cost automatically with zero manual work.

🔒 Storage access & security

MethodWhat it isWhen to use
Access KeysFull admin access (like a root password)Service connections — keep secret!
SAS TokenLimited time-bound token for specific accessShare a file externally for 24h
Azure AD / RBACIdentity-based access using rolesBest practice for users & managed identities
Public accessAnyone on the internet can readStatic websites, public assets only
✅ SAS token example
You need to let a client download a report for 24 hours. Generate a SAS token with: read-only permission, on the specific blob, expiring in 24h. The client gets a URL — they never see your account key, and the URL stops working after 24h automatically.
Quick Check — Module 04
You store log files that are rarely accessed after 90 days. You want to minimize storage costs. Which tier should you move them to using a lifecycle policy?
Module 05

Governance & Management

Organize, protect, and control costs across your Azure environment.

🏛 Azure management hierarchy

Azure resources are organized in a hierarchy. Permissions and policies applied at a higher level flow down to everything below.

📜 Azure Policy

Policies let you automatically enforce rules across your Azure environment. When a resource doesn't comply, it can be flagged or blocked from being created.

EffectWhat it does
DenyBlocks non-compliant resources from being created
AuditLogs non-compliant resources but still allows them
AppendAutomatically adds properties (e.g. a tag)
DeployIfNotExistsDeploys a related resource if it doesn't exist
✅ Policy example
Policy: "Allowed virtual machine SKUs = Standard_D2, Standard_D4 only."
Effect: Deny.

If a developer tries to create an expensive Standard_E64 VM in production — Azure blocks it automatically, before any resource is created.

🔒 Resource Locks

Locks prevent accidental deletion or modification of critical resources. They apply to ALL users, including Owners.

🗑
CanNotDelete
Nobody can delete the resource. Read and modify are still allowed.
📖
ReadOnly
Nobody can delete or modify. Read-only. Strong protection.
⚠️ Exam trap
Even an Owner cannot delete a resource with a CanNotDelete lock. They must first remove the lock, then delete. This is a deliberate safeguard — and a common exam trick.

🏷 Tags & Cost Management

Tags

Tags are name/value pairs you attach to resources. Used for cost allocation, searching, and automation.

Examples: Environment: Production, Team: DevOps, CostCenter: 4521

⚠️ Tags are NOT inherited by child resources by default. Use Azure Policy to enforce tagging.

Cost management tools

ToolPurpose
Cost AnalysisVisual breakdown of what you're spending, filtered by tags/resource type/time
BudgetsSet a spending limit — get alerts at 80%, 100%, etc.
Azure AdvisorAI recommendations for cost savings, performance, security, reliability
ReservationsPay 1–3 years upfront for up to 72% discount on VMs
